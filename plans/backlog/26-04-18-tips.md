# Tips (Propinas)

## Goal

Allow waiters to record tips on an order so they show on the bill, in cash closing, and admin reports.

## When to ask

**Recommended: on the bill view (Screen 9), before payment confirmation.**

Reasoning:
- Waiter already has the customer's attention and the paper bill in hand.
- Tip varies with the customer's decision at payment time — can't be predicted earlier.
- Pre-payment entry makes the charged amount accurate on the first try (no edit-after-close).
- Matches real-world Mexican restaurant flow: print bill → ask "¿Desea agregar propina?" → charge total + tip.

Rejected alternatives:
- *On item add*: premature, tip isn't per-item.
- *After payment*: forces reopen, contradicts "closed is terminal" decision.
- *Implicit percentage*: Mexican norm is customer-chosen, 10-15% suggestion. Forcing breaks UX.

## UX sketch

On bill view, above payment methods:

```
Propina
[ $0 ] [ 10% ] [ 15% ] [ 20% ] [ Otro: $___ ]
```

- Buttons set `tip_cents` on the order.
- "Otro" reveals input.
- Total updates live (Stimulus).
- Method radios below, confirm pays total + tip.

## Data model

- Add `tip_cents` integer to `orders` (default 0, non-null).
- `Order#grand_total_cents = total_cents + tip_cents`.
- `Payment.amount_cents` still covers grand total (no separate tip payment).
- `Order#fully_paid?` compares against `grand_total_cents`.

Open question: do tips appear as a separate line in Corte de Caja? Likely yes — admins track tip pool separately.

## Scope

**In:**
- `tip_cents` on Order
- Preset buttons + custom input on bill
- Bill view shows Propina line and Grand Total
- Payment flow uses grand total

**Out (follow-ups):**
- Tip pool distribution reports
- Per-waiter tip tracking (could come free if we query payments by user)
- Split tip across multiple payments

## Done when

- Waiter picks or enters tip on bill view
- Tip shown on printed bill and closed confirmation
- Payment charges total + tip
- Admin dashboard / cash closing aggregates tips
- Tests cover zero-tip, preset, custom, and grand total math
